# Engineering Principles

This repository follows the public portfolio standard used across Abdulelah Alkhathami's flagship AI projects.

## Principles

- Use synthetic or public-safe data in security prototypes.
- Prefer explainable risk signals before opaque model claims.
- Document privacy, threat-model, and production gaps early.
- Keep dashboards tied to observable backend state.
- Avoid implying production security readiness without validation.

## Applied Here

- The prototype uses synthetic behavior scenarios.
- Suspicious activity is explained through clear rules.
- In-memory state and permissive CORS are documented as prototype limitations.
- Future work calls out persistence, tests, privacy review, and threat modeling.
